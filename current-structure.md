# 현재 사이트 학습 구조 (있는 그대로)

> 이 문서는 코드베이스(2026-06 기준)를 분석해 **현재 구현되어 있는 상태만** 객관적으로 정리한 것입니다.
> 평가나 개선 제안은 포함하지 않습니다.

사이트는 설치·서버 없이 정적 파일로 동작하는 멀티페이지 웹사이트입니다. 모든 페이지는 공용 셸(`js/ui.js`)을 불러와 상단 내비게이션을 자동 구성하고, 데이터는 `js/store.js`(기본 로컬 저장, 설정 시 클라우드)에 저장됩니다.

---

## ① 학습 항목 · 기능 목록

### A. 만들기(창작) 스튜디오
- **색 군집 스튜디오 (`studio-color.html`)**
  - 이미지 입력: 명화 고르기(실제 퍼블릭 도메인 작품, 오프라인 시 절차적 대체) · 파일 업로드 · 드래그앤드롭
  - 분석: K-means 색 군집화(K 개수, 색공간 RGB/LAB, 샘플링 무작위/밝은영역/어두운영역/윤곽)
  - 팔레트 표시 + 색 분석 차트(도넛 · 가로막대 · 세로막대 · 트리맵 · 색상환 · 거품 · 히트맵)
  - 점(입자): N(점 개수), 점 크기, 색 모드(대표색/원본색/명암)
  - 표현 렌즈: 모자이크 격자 · 구조(에지=윤곽 선묘 / 구도=삼분할·시선 무게중심)
  - 움직임: 표현 모드(점/점+선/3D 조각), 움직임 기준(원위치 유지/자유 이동), 복귀력, 진동, 잔상, 발광, 자유 세부(떠돎·소용돌이·중심 인력), 3D 세부(회전축·회전 속도·깊이)
  - 인터랙션: 마우스(밀기/끌기/부풀리기/흩뿌리기·클릭 폭발), 마이크(볼륨·주파수 반응)
  - 프리셋(점묘화·살아있는 캔버스·소리로 연주·성운·3D 조각)
  - 내보내기: 이미지(PNG) · 영상(WebM) · 팔레트 색상코드 · 설정(JSON) · 분석 리포트/설계서(Markdown)
  - 윤리·리포트(작품명·작가·출처·제작자·의도) 기록
  - AI 감상 코치 · 작업노트 저장 · 갤러리 전시
  - 각 옵션 ⓘ 설명 팝업, K-means/표현 렌즈 설명 모달
- **데이터 점 스튜디오 (`studio-data.html`)**
  - 데이터 입력: 사회문제 샘플(기후·대기오염·불평등·지역소멸·동물권·감정) · CSV 업로드 · 직접 입력 · 양식 받기
  - 다중 열 자동 인식(수치/범주), 필드 칩 표시
  - 매핑 설계: 크기·속도·방향·밀도·투명도·형태(●▲■◆)·색(그라데이션/범주별/단색, 라벨별 색·형태)
  - 배치(레이아웃): 시간축·원형·격자·값 산포
  - 움직임 방식: 제자리 진동·궤도·파동·분출, 기본 속도·진동·잔상
  - 규칙 A/B 저장·비교 전환
  - 데이터 선택 기록(무엇을 셌고 무엇을 뺐는지)
  - AI 감상 코치 · 이미지 저장 · 작업노트 저장 · 갤러리 전시
- **소리를 데이터로 (`studio-sound.html`)**
  - 입력: 마이크 녹음 · 오디오 파일 · 샘플("아빠의 한 달 코골이" 상황별 이야기)
  - 특징 추출(프레임 단위): 음량 · 저음 · 중음 · 고음 · 날카로움(스펙트럼 무게중심) · 변화(spectral flux)
  - 시계열 시각화(동적 범례)
  - CSV 내보내기 · 데이터 점 스튜디오로 전송
- **내 삶을 데이터로 (`studio-life.html`)**
  - 사진(여러 장) → 밝기·대비·따뜻함·생생함(채도)·다양함(색 엔트로피)·북적임(에지 밀도)·주색 측정
  - 챗봇 대화 붙여넣기 → 순서·화자·글자수·물음표 추출
  - 데이터 선택 기록 · CSV 내보내기 · 데이터 점 스튜디오로 전송
- **객체 감지 · AI의 눈 (`studio-object.html`)**
  - 이미지 입력: 사진 업로드 · 오프라인 예시 장면 · 명화로 시험
  - 브라우저 AI(TF.js COCO-SSD)로 사물 감지(박스·라벨·신뢰도), 모델 지연 로드·오프라인 폴백
  - 감지 요약 · "AI의 눈" 한계 비평
  - CSV 내보내기 · 데이터 점 스튜디오로 전송
- **사회문제 프로젝트 (`project.html`)**
  - 공공데이터를 찾아 정제해 CSV로 만들고 데이터 점 스튜디오로 전송 · 작업노트 저장

### B. 분석 · 학습 도구
- **알고리즘 분석실 (`lab.html`)**: 한 그림을 7가지 알고리즘으로 분석 — ① K-means 색 군집 ② 색 히스토그램 ③ 명도·노탄 ④ 에지(Sobel) ⑤ 중앙값 분할 ⑥ 격자 평균·모자이크 ⑦ 구도 분석. 각 분석에 차트 + 재창조 + ⓘ설명 + "놓친 것" + 해석 메모. 재창조 이미지를 색 군집 스튜디오로 보내기.
- **알고리즘 배움터 (`learn.html`)**: 알고리즘 개념과 데이터→점 매핑 개념을 쉬운→깊은 단계로 설명.
- **데이터 비평 읽기 (`critique.html`)**: 차트의 '수사'를 질문 3층위(사실→해석→가치)로 비평, 성찰 저장.
- **학습 여정 지도 (`journey.html`)**: 4단계 학습 흐름·축·평가 영역 안내.

### C. 나누기 · 기록
- **전시 갤러리 (`gallery.html`)**: 전시된 작품 감상(Player 재생) + 또래 피드백.
- **키오스크 전시 (`exhibit.html`)**: 큰 화면 슬라이드쇼 + 작품별 QR(→ 작품 단독 페이지).
- **작품 단독 페이지 (`work.html?id=…`)**: 저장된 작품을 다시 재생 + 감상·비평(펠드먼 4단계). QR/갤러리로 진입.
- **분석 퀴즈 (`quiz.html`)**: 그림을 분석해 출제하고 친구 문제를 풀어 겨루기.
- **작업 노트 (`notes.html`)**: 과정·버전·성찰 기록.

### D. 교사
- **교사 대시보드 (`admin.html`)**: 전체 작품/노트 열람·취합·통계(교사 게이트).

### E. 로그인
- **로그인/시작 (`index.html`)**: 이름·반/번호·PIN·표시 이름·역할(학생/교사)로 계정 생성·로그인. 4단계 여정 미니맵·기능 안내. 로그인 없이 둘러보기 가능(저장·전시는 로그인 필요).

### F. 공용 모듈(전 페이지 공유)
- `ui.js`(헤더 내비·ⓘ 설명 모달·토스트·콜아웃·아이디어 바·푸터) · `auth.js`(로그인/세션) · `store.js`(저장: 로컬/클라우드) · `explain.js`(ⓘ 설명 콘텐츠) · `guide.js`(아이디어 콘텐츠) · `coach.js`+`ratelimit.js`(AI 감상 코치) · `player.js`(작품 재생) · `qr.js`(QR) · `docent.js`(도슨트) · `analysis.js`·`kmeans.js`·`particles.js`·`charts.js`·`audio.js`·`harmony.js`(색 스튜디오·분석실 엔진) · `app.js`·`studio-color-glue.js`(색 스튜디오) · `studio-data.js`·`studio-sound.js`·`studio-life.js`·`studio-object.js`·`lab.js`·`critique.js`·`project.js`·`quiz.js`·`work.js`·`exhibit.js`·`admin-gate.js`(각 페이지).

## ② 화면(페이지) 구분

총 18개 HTML 페이지. 상단 내비게이션(`ui.js`)은 4개 그룹으로 묶여 있습니다.

| 그룹 | 페이지 |
|---|---|
| **만들기** | 색 군집 스튜디오 · 데이터 점 스튜디오 · 소리를 데이터로 · 내 삶을 데이터로 · 사회문제 프로젝트 · 객체 감지(AI의 눈) · 알고리즘 분석실 |
| **배우기** | 알고리즘 배움터 · 데이터 비평 읽기 · 학습 여정 지도 |
| **나누기** | 전시 갤러리 · 키오스크 전시 · 분석 퀴즈 · 작업 노트 |
| **교사** | 교사 대시보드 |
| (내비 밖) | 로그인(`index.html`) · 작품 단독 페이지(`work.html`) |

- 허브(`hub.html`)는 내비 아래에서 카드로 한 번 더 묶입니다: 「만들기 — 4단계 심화 여정」(색/데이터 → 소리 → 내 삶 → 사회) / 「분석·렌즈·비평(도구)」(분석실 · 객체 감지 · 비평) / 「배우기·나누기」 / 「오늘의 탐구 문제」.
- 스튜디오 페이지는 좌측 작업 캔버스 + 우측 설정 패널(또는 카드 폼) 구조입니다. 색 군집 스튜디오는 공용 site.css/ui.js를 쓰지 않는 독립 셸(자체 모달·툴팁)이며 `studio-color-glue.js`로 사이트와 연동됩니다.

## ③ 학생 이동 순서 (현재 구현된 흐름)

1. **로그인** `index.html` — 로그인하면 기본적으로 `hub.html`로 이동(또는 `?next=` 로 지정된 페이지로).
2. **허브** `hub.html` — 「만들기 4단계 여정」과 도구/배우기/나누기로 갈라집니다.
3. **만들기 4단계(심화 여정)**:
   - 1단계: 색 군집 스튜디오 · 데이터 점 스튜디오
   - 2단계: 소리를 데이터로 → (전송) → 데이터 점 스튜디오
   - 3단계: 내 삶을 데이터로 → (전송) → 데이터 점 스튜디오
   - 4단계: 사회문제 프로젝트 → (전송) → 데이터 점 스튜디오
4. **분석실 경로**: 알고리즘 분석실 → (재창조 이미지 전송) → 색 군집 스튜디오.
5. **저장·전시**: 스튜디오에서 작품 저장/전시 → 전시 갤러리에서 감상·피드백 → (클릭/QR) 작품 단독 페이지(`work.html?id=`)에서 재생 + 비평.
6. **키오스크**: 키오스크 전시(`exhibit.html`)에서 슬라이드쇼 → 각 작품 QR → 작품 단독 페이지.
7. **로그인 필요 동작**(저장·전시·출제 등)은 비로그인 시 `index.html?next=<현재 페이지>`로 보낸 뒤, 로그인 후 원래 페이지로 돌아옵니다.
8. 상단 내비게이션·허브 카드로 어느 페이지에서나 다른 그룹(배우기/나누기/교사 등)으로 자유롭게 이동할 수 있습니다(고정된 강제 순서는 로그인 리다이렉트와 '전송→데이터 스튜디오' 외에는 없음).

## ④ 항목들이 서로 연결되는 방식

### 데이터 전달(브라우저 localStorage 키)
- `dn_data_incoming` — **소리·내 삶·객체 감지·사회문제 프로젝트 → 데이터 점 스튜디오**. 보내기를 누르면 CSV 페이로드를 저장하고 데이터 점 스튜디오로 이동, 데이터 점 스튜디오가 시작 시 이를 우선 로드합니다.
- `dn_studio_image`(+`dn_studio_image_title`) — **알고리즘 분석실 → 색 군집 스튜디오**. 재창조 이미지를 저장한 뒤 이동하면 색 스튜디오가 `loadImageURL`로 불러옵니다.
- `dn_quiz_done`(퀴즈 풀이 기록), `dn_exhibit_base`(키오스크 기준 URL), `dn_theme`(테마) 등 보조 상태.

### 저장 계층(`store.js`)
- `saveWork(...)` — 색·데이터 작품 저장(전시 플래그 포함). → **전시 갤러리·키오스크 전시·작품 단독 페이지·교사 대시보드**가 이를 읽음.
- `saveNote(...)` — 데이터/분석실/비평/사회문제 등의 해석·성찰 노트 저장. → **작업 노트·교사 대시보드**.
- `listWorks({exhibited:true})` — 키오스크/갤러리가 전시 작품 목록을 가져옴.
- `getWork(id)` — 작품 단독 페이지가 `work.html?id=<workId>`로 단일 작품을 가져옴.

### 작품 재생(`player.js`)
- 저장된 작품(설정 + 썸네일/원본)을 **갤러리·키오스크·작품 단독 페이지**에서 다시 '살아 움직이는' 화면으로 재생(데이터 작품=춤추는 점, 색 작품=점묘).

### QR 연결(`qr.js`)
- 키오스크 전시·작품 페이지에서 작품 단독 페이지(`work.html?id=`) QR을 생성 → 관람자가 휴대폰으로 진입해 감상·비평.

### 공통 연결
- **로그인/세션**(`auth.js`): 모든 보호 동작은 `Auth.current()` 확인 → 비로그인 시 `index.html?next=`로 리다이렉트.
- **AI 감상 코치**(`coach.js`+`ratelimit.js`): 색/데이터 스튜디오 등에서 현재 작품 맥락(팔레트·매핑·의도)을 읽어 질문을 제시(기본 오프라인, 설정 시 모델 호출).
- **ⓘ 설명 팝업**(`explain.js`+`ui.js`): 어느 페이지든 `[data-info="키"]` 요소를 누르면 쉬운→깊은→한계→아이디어 설명 모달(색 군집 스튜디오는 자체 모달 사용).
- **아이디어 로테이터**(`guide.js`): 페이지별 아이디어를 `mountIdeaBar`로 한 개씩 순환 표시.
- **공용 셸**(`ui.js`): 상단 내비게이션과 사이트 푸터를 모든 페이지에 주입.
