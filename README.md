# 데이터의 눈으로 그림 읽기 — 분석에서 창작까지

> 서울미술고등학교 · AIEDAP · 2022 개정 교육과정(미술·정보) AI 융합수업 지원 **멀티페이지 수업 사이트**
> 두 수업을 하나의 스튜디오로: **「그림이 분해되어 다시 연주되다」**(색 군집) +
> **「데이터가 춤추는 점」**(데이터 매핑).

그림·데이터를 **여러 알고리즘으로 분석**하고, 학생이 **의도와 조작 설정**으로
인터랙티브 제너러티브 미디어아트로 **재창조**합니다. 설치·서버·계정 없이 `index.html`로 시작합니다.

## 사이트맵

```
index.html  (로그인 · 이름/반-번호/PIN)
      │
hub.html  학생 허브
   ├─ 만들기 : studio-color.html(색 군집·1단계) · studio-data.html(데이터 점·1단계) · studio-sound.html(소리를 데이터로·2단계) · studio-life.html(내 삶을 데이터로·3단계) · project.html(사회문제·4단계) · studio-object.html(객체 감지·AI 렌즈) · lab.html(분석실)
   ├─ 배우기 : learn.html (알고리즘 배움터 · 쉬운→깊은 설명)
   └─ 나누기 : gallery.html(전시 갤러리) · exhibit.html(키오스크 전시·QR) · quiz.html(분석 퀴즈) · notes.html(작업 노트)

work.html?id=…  작품 단독 페이지 (QR 진입 · 감상·비평 펠드먼 4단계)

admin.html  교사 대시보드 (전체 열람 · 취합 · 통계)

[공유 데이터 계층] store.js — 기본 로컬, config/cloud-config.js 설정 시 실시간 클라우드
[AI 감상 코치]   coach.js — 기본 오프라인(질문 생성), 선택 시 모델 호출(rate-limit 보호)
```

설명 아이콘 **ⓘ**(쉬운 예시 → 깊은 원리 → 놓친 것 → 아이디어)와 안내 **💡**가 모든 페이지 곳곳에 있습니다.
상단 우측 **🌗 버튼으로 밝은/어두운 테마**를 전환합니다(브라우저에 기억).

## 핵심 기능

### 1) 알고리즘 분석실 (`lab.html`) — 한 그림, 일곱 개의 눈
한 그림을 7가지로 분석하고, 각각 **차트 + 재창조 결과 + ⓘ설명 + “이 분석이 놓친 것” + 해석 메모**(→ 리포트로 모임).

| 알고리즘 | 재창조 | 비고 |
|---|---|---|
| ① K-means 색 군집화 | 대표색 점 치환 | 핵심 |
| ② 색 히스토그램 | 포스터화 | 핵심 |
| ③ 명도·노탄 | 흑백 명암지도 | 핵심 |
| ④ 에지(Sobel) | 선 드로잉 | 핵심 |
| ⑤ 중앙값 분할(Median Cut) | 또 다른 색 요약 | 심화 · K-means와 비교 |
| ⑥ 격자 평균·모자이크 | 픽셀아트 | 심화 |
| ⑦ 구도 분석 | 무게중심·삼분할 | 심화 |

**색 분석 차트 6종**: 비율 도넛 · 정렬 막대 · 색공간 산점도(군집 중심 표시) · RGB 채널 히스토그램 · 색상환 분포 · 명도 히스토그램/톤커브. 각 차트 옆 **해석 메모**가 자동으로 리포트(부록 A)로 모입니다.

**명화 = 실제 퍼블릭 도메인 작품(위키미디어 공용, 공교육용)**: 고흐 별이 빛나는 밤 · 쇠라 그랑드자트섬의 일요일(점묘) · 모네 인상·해돋이 · 호쿠사이 큰 파도 · 다 빈치 모나리자 · 베르메르 진주 귀고리 소녀 · 클림트 키스 · 뭉크 절규 · 칸딘스키 구성 7 · 몬드리안 구성. 인터넷이 없으면 비슷한 절차적 그림으로 자동 대체. 사조와 분석을 잇습니다.

### 2) 색 군집 스튜디오 (`studio-color.html`)
명화를 K-means로 분석 → 대표색 점으로 치환 → 마우스(밀기/끌기/부풀리기/흩뿌리기·클릭 폭발)와
마이크(볼륨→진동·확산·크기, 주파수→밝기별 색 군집 반응)로 ‘연주’. 2D 점/선 + **3D 조각**(밝기→깊이).
*기존 단일 파일 도구를 그대로 편입 — 사이트 없이도 단독·오프라인으로 동작(백업).*

### 3) 데이터 점 스튜디오 (`studio-data.html`)
우리 데이터(감정·기온·걸음·소음/직접 입력/CSV)를 점의 **크기·속도·방향(벡터)·밀도·투명도·형태(△○□)**로 변환해 움직이는 작품으로.
매핑 규칙 **2안(A/B) 비교**, 의도·근거를 채워야 전시 가능(근거가 먼저).
**데이터 선택 기록(제작 문서)**: 감각먼저 → 무엇을 셌나/일부러 뺐나(숨긴 선택) → 왜 이 척도·색인가 → 데이터가 놓친 ‘내 삶의 진실’(메타비평). 〈Dear Data〉·데이터 휴머니즘을 ⓘ로 연결 — 데이터를 ‘비평 대상이자 창작 재료’로.

### 4) 배우기·나누기
- **배움터**: 7개 알고리즘 + 데이터 매핑 개념을 쉬운→깊은 설명으로. 미술사 다리(쇠라·로스코·옵아트·제너러티브).
- **갤러리**: 작품 감상 + **구조화된 피드백(의도 전달/근거/인터랙션/윤리)** → 받은 학생이 2차 개선(순환).
- **작업 노트**: 작업·버전·성찰 타임라인. “AI가 도운 것 / 내가 결정한 것” 기록.
- **분석 퀴즈(`quiz.html`)**: 그림을 분석해 ‘지배색·색채조화·색온도·구도’ 문제를 자동/직접 출제 → 친구가 풀어 점수·순위. 조형 요소 태그·해설로 감상·분석을 게임처럼.

### 5) 전시·발표 모드 — 키오스크 · QR · 감상·비평
실제 전시회를 위한 흐름입니다. **창작 → 전시 → 감상 → 비평**이 하나로 이어집니다.
- **키오스크 슬라이드쇼(`exhibit.html`)**: 큰 화면용 전체화면 전시. 작품 + 작가노트(의도·데이터·알고리즘) + **작품별 QR**을 자동으로 넘기며 보여줍니다(←/→·Space·F 단축키, 자동 넘김).
- **QR → 작품 단독 페이지(`work.html`)**: 관람객이 휴대폰으로 QR을 찍으면 그 작품 페이지가 열리고, **감상·비평을 직접 남깁니다**(로그인 없이 관람객 이름으로 가능).
- **감상·비평은 미술비평 표준 ‘펠드먼 4단계’**: ① 기술(무엇이 보이나) → ② 분석(색·명도·구도) → ③ 해석(의미·의도) → ④ 평가(근거 있는 판단). 가벼운 ‘별점+한마디’도 가능.
- **도슨트(작품 해설) 자동 생성**: 의도·데이터·알고리즘 설정을 읽어 한국어 해설을 만들어 작품 페이지·키오스크에 표시합니다.
- **작가의 과정·성찰 노출**: 학생의 성찰·분석 메모를 작품 페이지(키오스크 teaser)에 함께 전시 — 학습 과정도 작품의 일부.
- **큐레이션(교사)**: 교사 대시보드에서 작품마다 **전시 순서·테마**를 지정 → 키오스크가 순서대로 보여주고 테마별로 필터합니다.
- QR 주소는 자동(현재 접속 주소)이며, 배포/교실 LAN 주소로 바꿀 수 있습니다(🔗 버튼). 같은 네트워크/링크면 관람객 비평이 실시간(클라우드)으로 모입니다.

> 🔒 **교사 대시보드**(`admin.html`)는 관리자 비밀번호로 잠겨 있습니다(관리자: `sooremond@gmail.com`). 정적 사이트의 *가벼운 잠금*(클라이언트 측, SHA‑256 해시 비교)이므로 강한 보안이 아니며, 민감정보는 올리지 마세요.

### 6) AI 감상 코치 — ‘답’이 아니라 ‘질문’
팔레트·K·N·규칙·학생 의도를 읽어 **근거를 캐묻는 질문**을 생성합니다(‘AI 맹신 금지’ 가치).
- **기본: 완전 오프라인**(인터넷·키 불필요). 8명이 동시에 써도 오류 없음.
- **선택: 실제 모델 호출** — 프리 티어 한도(분당 요청 5·입력 10K·출력 4K)를 `ratelimit.js`가 보호하고,
  한도 초과/네트워크 오류 시 **자동으로 오프라인 질문으로 폴백**.

### 가장 쉽게
`index.html`을 브라우저로 엽니다(인터넷 불필요). p5.js는 `vendor/`에 동봉되어 **오프라인 동작**.

```bash

```

`.github/workflows/pages.yml`가 푸시할 때마다 자동 배포합니다. 발행 주소(프로젝트 페이지):
**`https://seoulartai1-cell.github.io/DATA2026SEOULART/`**
- 워크플로가 Pages 자동 활성화를 시도합니다. 만약 막히면 한 번만 **Settings → Pages → Source = “GitHub Actions”** 로 지정하세요.
- HTTPS라 마이크·녹화·클라우드가 정상 동작하고, 키오스크 QR도 이 주소를 가리킵니다.

## 실시간 클라우드 공유 — 한 링크에서 학급 전체 [연동됨]

`config/cloud-config.js`가 Firebase 프로젝트 **`data-art-2026`**(Firestore)에 연결되어 있어,
모든 페이지가 자동으로 실시간 공유 모드로 동작합니다. 설정 전/네트워크 불가 시에는 **로컬 저장으로 자동 폴백**합니다.

**남은 1회 설정(교사) — Firestore 보안 규칙 게시:**
1. [Firebase 콘솔](https://console.firebase.google.com) → `data-art-2026` → **Firestore Database → 규칙(Rules)** 탭.
2. `config/firestore.rules` 내용을 붙여넣고 **게시(Publish)**.
   - (현재 데이터베이스는 ‘잠금’ 상태라 게시 전에는 읽기/쓰기가 거부됩니다 → 자동으로 로컬 폴백.)
3. 끝! 같은 링크를 연 학생들의 작품·피드백·노트가 실시간으로 공유됩니다.

- **강제 로컬 전환**(점검·시험용): 주소 끝에 `?local=1` (예: `gallery.html?local=1`),
  또는 콘솔에서 `localStorage.setItem('dn_cloud_off','1')` 후 새로고침.
- 클라우드를 안 써도 **교사 대시보드 ‘취합’**으로 학생 JSON을 병합해 모을 수 있습니다.

> ⚠ **보안/개인정보**: Firebase 웹 `apiKey`는 비밀이 아니라 공개 식별자입니다(노출 정상). 실제 보안은
> `config/firestore.rules`로 합니다. 이 규칙은 *수업 기간 한정* 설정이며, 단원 종료 후 데이터 내보내기 →
> 프로젝트 정리를 권장합니다. PIN 4자리는 ‘실수 방지 잠금’ 수준이고, 미성년자 정보이므로 학교 방침을 우선하세요.
> 마이크 음성은 저장하지 않습니다.

## (선택) 실제 AI 모델 코치 켜기 — 교사 대시보드에서

**교사 대시보드(`admin.html`) → ‘🤖 AI 감상 코치 설정’** 카드에서 켭니다(코드 수정 불필요):
1. ‘실제 모델 사용’ 체크 → 방식·모델 선택 → **API 키 입력** → ‘설정 저장’.
2. ‘연결 테스트’로 실제 응답 여부 확인(실패 시 자동 오프라인).

- 키는 **그 브라우저(localStorage `dn_coach_cfg`)에만** 저장되고 **저장소·서버로 전송되지 않습니다**.
- 무료 한도(분당 요청 5·입력 10K·출력 4K)는 `ratelimit.js` 큐가 보호하고, **한도 초과/오류 시 자동으로 오프라인 질문으로 폴백** → 8명이 동시에 써도 학생 화면엔 오류가 안 납니다.
- **방식**: *직접 호출*(키가 그 브라우저에 노출 → 교사 기기 전용 권장) / *프록시*(작은 서버로 키 숨김·중앙 한도 관리 → 학생 공용 시 권장).
- 8명이 한 키를 공유하면 분당 5요청을 나눠 쓰므로, **오프라인 코치를 주력**으로 두고 실제 모델은 보조로 쓰는 것을 권장합니다.
